# Code of Conduct

## Our Standard

This project is intended to be technically serious, respectful, and useful.
Participation in CortexOS means contributing to a space where people can discuss
architecture, bugs, tradeoffs, and product decisions without hostility,
harassment, or performative noise.

Examples of behavior that contributes to a healthy environment include:

- giving technical feedback with evidence and clarity
- discussing tradeoffs without personal attacks
- assuming good faith while still holding a high engineering bar
- helping new contributors understand the repo and its constraints
- respecting different levels of experience and different communication styles

Examples of unacceptable behavior include:

- harassment, insults, or threats
- personal attacks instead of technical arguments
- publishing private information without permission
- spam, trolling, or intentionally derailing technical threads
- low-effort disruption that makes maintenance harder for others

## Scope

This Code of Conduct applies to:

- GitHub issues
- pull requests
- discussions
- documentation feedback
- other public spaces associated with this repository

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull
requests, commits, code, wiki edits, discussions, and other contributions that
do not align with this Code of Conduct.

## Reporting

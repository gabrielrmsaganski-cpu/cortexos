# Security Policy

## Supported Versions

Security fixes are currently applied against the latest version on `main` and
the latest tagged release.

## Reporting a Vulnerability

Please do not open a public issue for a security vulnerability.

Use one of these paths:

1. Open a private vulnerability report through GitHub Security Advisories if
   that option is available on the repository.
2. If private reporting is not available, contact the maintainer through the
   GitHub profile:
   - https://github.com/gabrielrmsaganski-cpu

When reporting, include:

- the affected component
- impact and threat model
- reproduction steps
- whether exploitation requires auth or local access
- any suggested mitigation

## Response Model

Best effort:

- acknowledgement after review
- reproduction and triage
- fix or mitigation if the report is valid
- coordinated disclosure when appropriate

No SLA is promised, but legitimate reports will be taken seriously.
